---
title: "Compliance Scrubbing"
description: "How Hypr intercepts and redacts PII, API secrets, and credentials before they reach any LLM provider endpoint"
---

Every message Hypr sends to an LLM inference provider passes through a streaming compliance scrubber. The scrubber runs at the transport boundary inside `LLMClient.sendRequest()` — after the agent constructs its message payload but before any network call is made.

No secret ever leaves the host machine.

## How It Works

The scrubber walks the entire message tree recursively — including nested content block arrays inside assistant messages — and applies ten pattern rules to every string value it encounters.

```typescript
const { messages: safeMessages, totalHits } = scrubMessages(messages);
if (totalHits > 0) {
  process.stderr.write(`[scrubber] Redacted ${totalHits} secret(s) from outbound payload\n`);
}
```

Matched values are not deleted. They are replaced with **deterministic SHA-256 hash placeholders**:

```
sk-abcdef1234567890abcdef → [REDACTED:OPENAI_KEY:3f9a1c04b872]
AKIAIOSFODNN7EXAMPLE     → [REDACTED:AWS_ACCESS_KEY:d4e2f0a1b3c7]
```

The hash is derived from the matched secret value using SHA-256, truncated to 12 hex characters. The same secret always produces the same placeholder — so if a token appears in multiple messages, the LLM still receives a coherent, consistent redaction token rather than noise.

## Pattern Coverage

| Rule | Detects |
|---|---|
| `aws_access_key` | `AKIA...` 20-char IAM key IDs |
| `aws_secret_key` | 40-char base64 AWS secret access keys |
| `bearer_token` | `Authorization: Bearer <token>` headers |
| `generic_api_key` | `api_key=`, `access_token=`, `auth_token=` assignments |
| `github_pat` | `ghp_...` and `github_pat_...` personal access tokens |
| `openai_key` | `sk-...` OpenAI API keys |
| `anthropic_key` | `sk-ant-...` Anthropic API keys |
| `ssh_private_key` | PEM-encoded RSA, EC, and OpenSSH private key blocks |
| `env_secret` | `KEY=`, `TOKEN=`, `SECRET=`, `PASSWORD=` shell/env assignments |
| `connection_string` | `mongodb://`, `postgres://`, `mysql://`, `redis://` URIs |
| `absolute_path` | `/home/`, `/Users/`, `C:\Users\`, `D:\` local paths |

## Performance

The scrubber is benchmarked against 15,000-token synthetic payloads with seed secrets injected across all rule categories. Median scrub time: **under 10ms**. The PRD acceptance ceiling is 40ms — Hypr stays well beneath it.

## Stderr Reporting

When secrets are intercepted, a count is written to `stderr` inline with execution. This intentionally avoids polluting the main OpenTUI canvas stdout buffer:

```
[scrubber] Redacted 3 secret(s) from outbound payload
```

The message is diagnostic-only. The sanitized payload continues to the provider normally.

## Air-Gapped Telemetry

For enterprise deployments that need to disable all outbound operational metrics, set the `HYPR_TELEMETRY` environment variable at compile time:

```bash
HYPR_TELEMETRY=false bun run build
```

When this flag is set, Hypr does not emit any usage telemetry. Combined with the compliance scrubber, this gives air-gapped environments a provable zero-exfiltration execution guarantee.
